# 阶段04：前端工作台替换 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: 先用 `superpowers:using-git-worktrees` 创建隔离工作区，再用 `superpowers:subagent-driven-development`（推荐）或 `superpowers:executing-plans` 执行。本阶段只消费已经落地的 `/api/research-platform` 和 `research_platform` 频道，不反向发明后端协议。

**Goal:** 用“数据收集 / 模型训练”双页工作台替换现有趋势研究主流程，同时保留旧 `TrendResearchPanel` 入口壳，完成前端 API 接入、WebSocket 订阅、页面组件与测试。

**Architecture:** 保留“分析中心 > 趋势研究”入口，但在内部切换到新的 `ResearchPlatformPanel` 工作台。前端状态拆成 `useResearchCollectionWorkspace.mjs` 与 `useResearchTrainingWorkspace.mjs` 两个独立编排层，由对象中心组件消费 `session`、`dataset_manifest`、`training_run` 三类核心对象。

**Tech Stack:** Vue 3、现有 Electron + Vite 前端、Axios API service、现有 WebSocket client、Node built-in test runner。

**Frozen UI Contract Locks:**
- 数据收集页必须展示：当前活跃会话、历史会话列表、每个会话的起止时间、`planned_duration_sec`、`stop_reason`、`coverage_ratio`、`valid_second_count`、`missing_second_count`、`book_stale_second_count`、`state_stale_second_count`、会话内 1s 价格/成交/盘口特征曲线、`inference-ready`/`training-ready` 样本数、按 stratum 的覆盖分布。
- 模型训练页必须把 `dataset_manifest` 作为一等对象展示：`included_session_ids`、`feature_recipe_version`、`label_definition_version`、`integrity_policy_version`、`sampling_stride_sec`、`split_definition_version`、`embargo_sec`、`weighting_version`、`deployment_target_version`、`target_census_policy_version`、`target_window_policy_version`、`shift_state_definition_version`、`strata_definition_version`、`strata_fit_ref`、`weight_estimator_version`、`weight_fit_ref`、`domain_classifier_version`、`domain_classifier_fit_ref`、`shift_diagnostic_version`、`dataset_status`、`support_overlap_result`、`evaluation_protocol_version`、`score_definition_version`、`prerank_definition_version`、`policy_definition_version`、`policy_parameter_ref`、`decision_utility_version`、`utility_parameter_ref`、`execution_assumption_version`、`refit_policy_version`、`multiple_comparison_version`、`target_census_count`、`train/val/test` 名义样本数和 `n_eff`；同时必须展示 `regime_schema`、`n_eff_summary.sequence_definitions` 与 protocol whitelist 校验通过状态。
- 训练结果展示必须支持 `weighted` / `unweighted` diagnostics 切换、baseline 对比、主评分、校准、sharpness、regime 分层指标、block bootstrap 区间、冻结 policy 下的决策层指标；并显式展示 `weight_normalization.formula`、`multivariate_rank_histogram.rank_scheme = band_depth_rank_v1`、DRE 的 `class_prior_pi/sampling_scheme/calibration_method`、以及 `split_artifact` 中每个 outer origin 的 `inner_validation_folds`。
- WebSocket 事件只使用对象中心事件：`session_updated`、`census_updated`、`dataset_manifest_created`、`dataset_preview_updated`、`training_run_updated`。

export function useResearchCollectionWorkspace() {
  const sessions = ref([]);
  const activeSession = ref(null);
  const selectedSessionId = ref('');
  const sessionCharts = ref({ price: [], trade: [], book: [] });
  const sessionCoverage = ref({
    coverageRatio: 0,
    validSecondCount: 0,
    missingSecondCount: 0,
    bookStaleSecondCount: 0,
    stateStaleSecondCount: 0,
    inferenceReadyCount: 0,
    trainingReadyCount: 0,
    stratumCoverage: [],
  });
  const censusStatus = ref(null);

  const loadSessions = async () => {
    const response = await api.getResearchPlatformSessions({ limit: 50 });
    sessions.value = response.sessions || [];
  };

  const loadSessionDetail = async (sessionId) => {
    const response = await api.getResearchPlatformSessionDetail(sessionId);
    activeSession.value = response.session;
    sessionCharts.value = response.session.charts || { price: [], trade: [], book: [] };
    sessionCoverage.value = response.session.coverage || sessionCoverage.value;
  };

  const loadCensusStatus = async () => {
    const response = await api.getResearchPlatformCensusStatus();
    censusStatus.value = response.status;
  };

  return {
    sessions,
    activeSession,
    selectedSessionId,
    sessionCharts,
    sessionCoverage,
    censusStatus,
    loadSessions,
    loadSessionDetail,
    loadCensusStatus,
  };
}
```

export function useResearchTrainingWorkspace() {
  const datasets = ref([]);
  const trainingRuns = ref([]);
  const selectedDatasetId = ref('');
  const selectedRunId = ref('');
  const selectedDatasetManifest = ref(null);
  const selectedDatasetPreview = ref(null);
  const selectedRunDetail = ref(null);
  const selectedRunArtifacts = ref(null);
  const diagnosticsMode = ref('weighted');
  const baselineComparison = ref(null);
  const selectedSplitArtifact = computed(() => selectedRunArtifacts.value?.split_artifact || null);
  const selectedRegimeSchema = computed(() => selectedDatasetPreview.value?.regime_schema || null);

  const loadDatasets = async () => {
    const response = await api.getResearchPlatformDatasets({ limit: 50 });
    datasets.value = response.datasets || [];
  };

  const loadDatasetDetail = async (datasetId) => {
    const [datasetResponse, previewResponse] = await Promise.all([
      api.getResearchPlatformDatasetDetail(datasetId),
      api.getResearchPlatformDatasetPreview(datasetId),
    ]);
    selectedDatasetManifest.value = datasetResponse.dataset;
    selectedDatasetPreview.value = previewResponse.preview;
  };

  const loadTrainingRuns = async (datasetId = '') => {
    const response = await api.getResearchPlatformTrainingRuns({ dataset_id: datasetId, limit: 50 });
    trainingRuns.value = response.training_runs || [];
  };

  const loadTrainingRun = async (runId) => {
    const response = await api.getResearchPlatformTrainingRun(runId);
    selectedRunDetail.value = response.training_run;
    selectedRunArtifacts.value = response.training_run.artifacts || null;
    baselineComparison.value = response.training_run.artifacts?.baseline_result || null;
  };

  return {
    datasets,
    trainingRuns,
    selectedDatasetId,
    selectedRunId,
    selectedDatasetManifest,
    selectedDatasetPreview,
    selectedRunDetail,
    selectedRunArtifacts,
    selectedSplitArtifact,
    selectedRegimeSchema,
    diagnosticsMode,
    baselineComparison,
    loadDatasets,
    loadDatasetDetail,
    loadTrainingRuns,
    loadTrainingRun,
  };
}
```
