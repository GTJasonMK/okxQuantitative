# OKX 永续趋势数据采集与统计分析设计

## 背景

本项目已经具备 OKX 行情抓取、本地 SQLite 存储、K 线同步、实时行情缓存和基础指标计算能力，但现有链路主要服务于图表展示、回测和交易接口，还没有形成一套面向“跨币种统一趋势结论”的数据研究体系。

本次设计目标不是直接堆叠更多指标，而是围绕固定白名单永续合约，建立一套统一的数据采集、离线标注、因子筛选和实时推断机制，使每个币种都能持续输出同口径的短线趋势分数、状态分类以及局部高低点概率。

## 目标

- 只覆盖固定白名单的 USDT 永续合约。
- 建立统一的多源数据采集机制，覆盖价格、逐笔、盘口、标记价格和合约状态。
- 用统计学方法先做离线高低点标注和因子筛选，再进入实时推断。
- 输出跨币种可比较的统一结果：
  - `trend_score`
  - `trend_state`
  - `top_probability`
  - `bottom_probability`
  - `confidence`

## 非目标

- 不在第一版覆盖现货或动态白名单。
- 不直接使用黑箱端到端大模型作为第一版主方案。
- 不把所有原始数据直接入模。
- 不引入 silent fallback、mock 数据或伪成功路径。

## 标的与数据源

研究对象固定为配置化永续白名单，每个 `instId` 单独采集、单独建模，但共享统一口径。

对每个币种建立四层公共市场数据采集：

1. 价格层
   - `tickers`
   - `1m candles`
   - `mark price`
   - `index price`

2. 成交层
   - `trades`
   - 聚合后的主动买卖成交额、成交笔数、大单占比

3. 盘口层
   - `books` 或有限档位深度
   - 买卖盘失衡、价差、微价格、深度坍塌速度

4. 合约状态层
   - `open interest`
   - `funding rate`
   - `premium / basis`
   - 合约静态信息

实时主链路优先使用 WebSocket，REST 只用于启动回补、断线补缺口和低频状态拉取。历史价格主表统一以 `1m K线` 为基础，多周期数据在本地聚合生成，避免重复向 OKX 拉多周期数据。

## 存储设计

现有 `candles`、`market_ticker_snapshots`、`market_recent_trades` 继续保留，但需要新增三类研究表：

- `feature_bars_1s`
  - 统一的每秒特征条，是研究与实时推断的共同输入
- `swing_labels`
  - 离线生成的趋势状态、高点、低点和距离标签
- `trend_inference`
  - 实时输出结果与数据质量字段

如果盘口原始事件需要保留，只保留必要字段，不长期存储大体积全量原始 payload。

所有表都必须带：

- `inst_id`
- `ts_exchange`
- `ts_local`
- `data_quality`

这样后续统计分析可以识别缺口、延迟和断线补数区间，避免把脏数据当真值。

## 统一时间轴与特征条

原始事件频率差异很大，不能直接做相关分析。所有实时数据先压到统一时间网格，第一版建议以 `1s` 为核心粒度。

每个 `1s` 特征条至少包含：

- `mid_price`
- `mark_price`
- `index_price`
- `return_1s / 5s / 30s`
- `spread_bps`
- `book_imbalance_5 / 20`
- `microprice_deviation`
- `signed_trade_notional`
- `trade_count`
- `large_trade_ratio`
- `realized_vol`
- `oi_delta`
- `basis_zscore`
- `funding_regime`

所有特征都要标准化成跨币种可比口径：

- 价格类使用 `bps`、收益率或 `z-score`
- 成交量类使用相对滚动中位数或分位数
- 深度类使用名义金额或失衡比例
- `OI`、`funding`、`basis` 使用变化率和滚动标准化值

## 离线标签定义

本设计不使用固定未来窗口来定义趋势，而采用自适应波段标注。主价格轴优先用 `mark price` 或 `mid price`，避免 `last price` 的单笔成交噪声。

离线波段识别采用 `directional change / volatility-adjusted zigzag`：

- 动态阈值 `theta_t = k * sigma_t`
- `sigma_t` 由过去滚动窗口的实现波动率、ATR 或绝对收益中位数估计
- 价格从局部低点反转并超过阈值时确认上升段
- 价格从局部高点反转并超过阈值时确认下降段

基于该标注体系生成以下标签：

- `trend_state`
  - `uptrend`
  - `downtrend`
  - `range`
- `swing_top_confirmed`
- `swing_bottom_confirmed`
- `time_to_top`
- `time_to_bottom`
- `excursion_up`
- `excursion_down`

这些标签是后续因子研究和实时模型训练的唯一真值来源。

## 因子候选池与统计筛选

采集层可以宽，但建模层必须严。所有候选因子先进入研究池，再经过三步筛选。

第一步是单因子有效性分析：

- 连续标签使用 `Spearman IC`
- 事件标签使用 `ROC-AUC`、`PR-AUC`、`Lift`
- 对多个提前量窗口分别评估，例如 `30s`、`60s`、`180s`

第二步是稳定性分析：

- 按币种分组，检查方向是否一致
- 按时间滚动窗口，检查是否经常翻符号
- 按市场状态分层，检查单边、震荡、高波动、低波动下是否稳定

第三步是去冗余：

- 因子相关矩阵
- `VIF`
- 层次聚类
- 每簇只保留一个最稳定的代表因子

只有同时满足以下条件的因子才进入正式模型：

- 在白名单多数币种上方向一致
- 在样本外窗口保持有效
- 至少一个领先窗口显著有效
- 与已选因子不高度重复

## 实时在线推断

在线推断基于统一 `1s` 特征条，每个币种维护一份滚动状态，按秒刷新。

在线层不只输出一个连续分数，而是组合“状态机 + 概率模型”：

- `range`
- `uptrend_candidate`
- `uptrend_confirmed`
- `top_warning`
- `downtrend_candidate`
- `downtrend_confirmed`
- `bottom_warning`

概率模型负责估计：

- `trend_score`
- `top_probability`
- `bottom_probability`

状态机负责减少抖动和约束状态切换，避免在微观噪声中频繁来回翻转。

每次实时输出都必须附带质量字段：

- `confidence`
- `data_freshness_ms`
- `feature_coverage`
- `missing_channels`
- `last_gap_seconds`

如果盘口、逐笔或合约状态流中断，只允许显式降低置信度，不允许静默伪造完整信号。

## 验证方法

验证分为离线研究验证和在线事件回放验证。

离线验证要求：

- 按时间切分，禁止随机打散
- 使用 `walk-forward` 验证
- 每个币种单独统计，再做跨币种汇总

在线回放验证要求：

- 回放历史事件流
- 重放实时特征更新和状态切换逻辑
- 统计信号延迟、误报率和抖动程度

核心评估指标包括：

- 趋势状态的 `macro F1` 和混淆矩阵
- 高低点概率的 `ROC-AUC`、`PR-AUC` 和校准曲线
- 信号后的 `MFE / MAE`
- 发出顶部或底部预警后的实际回撤或反弹分布
- 跨币种效果分布和中位数稳定性

同时必须与简单基线对比，例如：

- 只用收益率和成交量
- 只用 `RSI / MACD`
- 只用价格与 `OI delta`
- 简单 `zigzag` 反转规则

只有在多数币种、多个时间阶段都稳定优于这些基线，复杂采集与建模才有成立价值。

## 分阶段落地

建议按以下顺序实施：

1. 固定永续白名单配置
2. 打通 `WS 主采集 + REST 回补`
3. 落地 `feature_bars_1s`
4. 生成离线 `swing_labels`
5. 完成单因子有效性、稳定性和去冗余分析
6. 训练第一版轻量实时模型
7. 接入在线推断和可视化监控

第一版模型优先选择可解释方案，例如正则化逻辑回归、小型树模型或分层线性组合，不直接进入高复杂度黑箱模型。

## 风险与约束

- 高频盘口和逐笔数据会显著提高存储和回放成本，因此必须尽早做聚合和裁剪。
- 强平类公共事件不应被当作完整真值，只能作为风险辅助因子。
- REST 与 WebSocket 的语义和延迟并不相同，实时分析必须以 WebSocket 为主时钟。
- 当前仓库工作树较脏，后续实施时只能精确修改本次涉及文件，不能回滚无关改动。

## 结论

本设计的核心原则是：

先把 OKX 永续市场的多源公共数据按统一时间轴采全，构造成跨币种可比的标准化特征，再用自适应波段标注生成高质量离线真值，通过单因子有效性、稳定性和去冗余筛选出少量可泛化因子，最终以状态机加概率模型的方式，为固定白名单中的每个币种持续输出统一口径的趋势结论与高低点判断。
